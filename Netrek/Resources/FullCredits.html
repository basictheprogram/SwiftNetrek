Swift Netrek Client Credits

Open Source Software

The Swift Netrek Client is built on the shoulders of giants. Prior versions of Netrek, Jtrek, and Xtrek go back to the 1986 timeframe. I personally enjoyed playing Netrek on DecStation 3100's and NeXT workstations at CMU in 1990-1991. Without these prior implementations (as well as the servers, which the client uses) the Swift Netrek Client would not exist.

In accordance with the debt to open source software, using the Swift Netrek Client is free. In addition the source code will be made publically available on GitHub as soon as Apple approves the client for the MacOS App Store.

Programmer

The Swift Netrek client is written by Darrell Root for Network Mom LLC. Check out https://networkmom.net/netrek for more information.

Programming Language and environment

The Swift Netrek client is written in.......Swift! The new programming language from Apple. Chris Lattner, formerly of Apple, was the originator and leader of the Swift project. We are eternally grateful.

Netrek uses the new "Network" framework from Apple. This is a new API to originate TCP and UDP sockets. It is noticably easier to use than the old BSD socket API.

Netrek uses SpriteKit for displaying graphics in the tactical map.

Graphics

The planet icons were made by Darrell Root based on screenshots of the original COW client from the 1990's

The Kleptonian (greenfleet) and Orion (blueFleet) ship icons came originally came from Artist Pascal Gagnon via the Gytha Netrek Client under the Creative Commons Attribution License.

The Roman (redfleet), Federation(outlineFleet), and Independent (also outlineFleet) ship icons came from the MacTrek 1.5 client (written in Objective C in 2006) by Chris and Judith Lukassen. It is possible that some of these fleets (redfleet in particular) may also have originated from artist Pascal Gagnon.

Prior Netrek Client: Mac Trek

Chris and Judith Lukassen wrote Mac Trek in Objective C around 2006. While we did not directly use any of their source code in our final product, we read their code in detail. We learned much about both Netrek and programming from their code, and thank them.

Prior Netrek Client: Gytha

James Cameron (aka Quozl) wrote the Gytha netrek client in 2012. Implemented in Python, we used the code as a resource during our development. James Cameron helpfully answered several questions during our development.

Prior Netrek Client: BRMH

Developed in C in the 1990's we also used this client as a resource during development. There are too many developers of BRMH so we refer you to http://www.netrek.org/files/archive/BRMH/BRMH.html for more information.

Netrek Vanilla Server

We used the Netrek Vanilla Server (written in C, maintained by James Cameron) as a reference during development. We used packets.h from this distribution to specify a few Struct formats we could not specify in Swift. See https://github.com/quozl/netrek-server for more information.
